Описание

Блок button используется для создания различных типов кнопок и предоставляет возможности по их стилизации. Кроме этого, позволяет стилизовать ссылку <a href> в виде кнопки. Функциональность кастомной кнопки основана на нативной, которая скрыта. В MSIE8 и ниже кастомные кнопки деградируют до нативных кнопок <button>, кнопка-ссылка станет нативной ссылкой <a>.

Варианты использования кнопок

С помощью button можно создать следующие типы кнопок:

  • Обычная кнопка – применяется для любых кнопок веб-интерфейса, используется по умолчанию. Не отправляет данные на сервер. По умолчанию тема оформления кнопки обычная (модификатор theme со значением normal).
{
    block: 'button', // имя блока
    mods: {size: 'm'}, // поле модификаторов. Модификатор размера - обязательный
    content: 'Я.Button'  // текст на кнопке
}

  • Кнопка действия – предназначена для отправки данных на сервер (submit). Должна располагаться в форме.

    Чтобы кнопка стала кнопкой действия, во входном BEMJSON блока вам нужно добавить поле type со значением submit, {String} type – тип кнопки.

    Обычно кнопка-действия визуально выделяется на странице темой оформления (модификатор theme со значением action, необязательный). Если нужно, для submit можно использовать любую тему.

{
    tag: 'form',
    attrs: {action: '#'},
    content: {
        block: 'button',
        mods: {theme: 'action', size: 'm'},
        type: 'submit',
        name: 'my-submit',
        content: 'Я.Submit'
    }
}

Кнопке-действия и обычной можно устанавливать атрибуты нативной кнопки <button атрибуты>: name и value задаются в одноименных полях входного BEMJSON блока. accesskey, tabindex, autofocus – в зарезервированном поле attrs BEMJSON.

  • Кнопка-ссылка – разновидность обычной кнопки, для которой дополнительно задано поле {String} url.

    Применяется для перехода по ссылке. Кнопкой-ссылкой может стать любая кнопка с полем url. В выходном HTML такая кнопка будет преобразована в тэг <a атрибуты> </a>, поле url – в атрибут ссылки href.

    Кнопке-ссылке можно задать необязательные атрибуты нативной ссылки, которые преобразуются в соответствующие HTML-атрибуты тэга <a>: accesskey, charset, hreflang, media, name, rel, rev, tabindex, title, type, target — задаются в зарезервированном поле атрибутов attrs. target — можно задать и в одноименном поле BEMJSON блока.

{
    block: 'button',
    mods: {size: 'm'},
    url: 'ya.ru',
    content: 'Я.Ссылка'
}

  • Псевдо-кнопка – любая кнопка, которой задан модификатор псевдо pseudo со значением yes.

    При нажатии на псевдо-кнопку JavaScript запускает какое-либо действие (popup и прочее). При отсутствии JavaScript происходит переход по ссылке (для кнопки-ссылки).

    По умолчанию псевдо-кнопке устанавливается тема оформления псевдо – модификатор theme со значением pseudo. У псевдо-кнопки отсутствует визуальный эффект нажатия (текст не сдвигается при нажатии). Кнопка прозрачная с бордером и по размеру меньше обычной кнопки (модификатор theme со значением normal).

Для появления у псевдо-кнопки визуального эффекта нажатия (текст сдвигается на 1px вниз и добавляется прозрачная желтая заливка), нужно установить модификатор pseudo-pressed со значением yes.

Тема псевдо может использоваться без модификатора pseudo. Чаще всего псевдо используется для оформления ссылки в виде кнопки.

{
    block: 'button',
    url: '/',
    mods: {size: 'm', pseudo: 'yes'},
    content: 'Псевдо-кнопка'
}
{
    block: 'button',
    mods: {size: 's', pseudo: 'yes'},
    content: 'Псевдо-кнопка'
}
  • Кнопка со счетчиком – такой может стать любая кнопка: обычная, действия, ссылка, для которой дополнительно задан модификатор counter со значением yes. Кнопка со счетчиком используется для сбора различных статистических данных, с разной вероятностью показа на веб-странице. Вероятностью показа счётчика для конкретной кнопки управляет поле блока show-counter. При show-counter = 100 — показ счётчика гарантирован всегда. По умолчанию вероятность срабатывания счётчика - 10%.
{
    block: 'b-form',
    attrs: {action: '#', method: 'get'},
    content: [
        {
            content: [
                {
                    block: 'button',
                    mods: {size: 'm', counter: 'yes'},
                    counter: 123,
                    content: 'Я.Кнопка'
                },
                ' — со счётчиком с вероятностью показа по умолчанию'
            ]
        }
    ]
}

Содержимое кнопки

Кнопке можно задать произвольное содержимое в зарезервированном поле content. Значение по умолчанию для него не задано. Кроме текста, на кнопку можно добавить иконку.

Допустимы следующие варианты наполнения кнопки:

  • У кнопки поле content отсутствует или пусто {block: 'button'}. В этом случае кнопка схлопывается по высоте и ширине.

    Если задан только размер кнопки, то ее высота будет соответствовать высоте указанного размера.

  • Контент содержит только текст (нужно указывать размер для блока).

  • Кнопка содержит только иконку.

    В этом случае кнопке нужно добавить модификатор only-icon со значением yes и смиксовать элемент кнопки icon с блоком иконки image.

    image — блок из библиотеки islands-components, который служит контейнером для изображения (преобразуется в HTML-тэг <img>). Нужное изображение задается с помощью нового модификатора блока image (произвольная пара «ключ:значение»), для которого в CSS хранится путь к изображению.

  {
    block: 'button',
    mods: {size: 's', 'only-icon': 'yes'},
    content: {
        block: 'image',
        mix: [{block: 'button', elem: 'icon', mods: {'16': 'comment'}}],
        alt: ''
     }
}
  • Кнопка содержит иконку и текст
{
    block: 'button',
    mods: {size: 'm'},
    content: [
        {
             block: 'image',
             mix: [{block: 'button', elem: 'icon',  mods: {'16': 'settings'}}],
             alt: '/'
        },
        'Я.Кнопка'
   ]
}

Модификаторы

Темы оформления

Любой кнопке можно задать одну из следующих тем оформления (модификатор темы theme):

  • Обычная normal (используется по умолчанию) – кнопка со светло-серым градиентом, под кнопкой едва заметная тень.

  • Действия action – кнопка с желтой заливкой. Основное назначение этой темы – визуальное выделение кнопки, выполняющей главное действие (обработка данных), т.е кнопки-действия.

  • Псевдо pseudo – кнопка прозрачная со светло-серым контуром, без тени и по размеру меньше обычной кнопки.

  • Темно-серая dark-grey – кнопка темно-серого цвета. Используется только для обычной кнопки и кнопки-ссылки при размещении на темном фоне.

  • Без рамки clear – Кнопка с прозрачным фоном, полупрозрачным текстом и без рамки. При наведении курсора текст становится непрозрачным. Не используется в качестве элемента формы, поэтому статус disabled - запрещен.

Помимо тем, для стилизации кнопок используются следующие модификаторы:

  • shadow со значением yes

    Кнопка с большой полупрозрачной тенью или «летающая» кнопка. Используется для визуального выделения кнопки, размещенной на сложном фоне, например, над картой. По нажатию «приземляется».

  • side со значением left(или right) – кнопка, закругленная только с левого (правого) края.

  • round со значением yes

    Круглые кнопки обычно используются для проигрывания музыки. По умолчанию круглой кнопке добавляется модификатор only_icon со значением yes и выполняется микс с блоком иконки image, от которого приходит CSS. Иконки находятся в спрайте (в формате svg). Диаметр кнопки 32px.

  • arrow со значением down

    Кнопка со стрелкой вниз. Может использоваться, например, для реализации выпадающих списков – блок select.

Размеры кнопок

Для кнопок реализованы размеры s и m. Модификатор размера size – обязателен при объявлении блока. По умолчанию не определен.

Высота кнопки зависит от размера шрифта и line-height.

Исключение: псевдо-кнопка с темой оформления псевдо (модификатор themeсо значением pseudo) имеет меньшую высоту, чем обычная кнопка ( модификатор theme со значением normal).

{
    block: 'button',
    mods: {size: 's'},
    content: 'Я.Кнопка размером S'
}

Создание кастомного размера

Скопировать существующий CSS-файл с размерами и вставить в него необходимые новые параметры.

Состояния кнопок

Кнопки могут находиться в следующих состояниях:

  • Не активна, модификатор disabled.
  • В фокусе, модификатор focused.
  • Наведение курсором, модификатор hovered.
  • Нажата, модификатор pressed.

Для появления эффекта нажатия у псевдо-кнопки используется дополнительное состояние:

  • Псевдо-кнопка нажата, модификатор pseudo-pressed.

Модификаторы состояния кнопки устанавливаются со значением yes.

Все состояния обычной кнопки:

Если кнопка находится в состоянии в фокусе, по ней можно выполнить нажатие клавишей Space или Enter. Переход по контролам формы осуществляется клавишей Tab.

Кнопка может находиться в нескольких состояниях одновременно. Например, в фокусе + наведение курсором + нажата.

В состоянии не активна кнопка отображается с прозрачностью и недоступна для действий пользователя. Такая кнопка не может получить фокус путем нажатия на клавишу Tab, мышью или другими способами. Тем не менее, такое состояние кнопки можно изменять программно. Значение не активной кнопки не передается на сервер.

С помощью модификатора focus со значением no можно запретить визуальное отображение фокуса у кнопки, и событие focus не возникает.

Значения по умолчанию

  • Тип кнопки: обычная <button type="button">.
  • Тема оформления: обычная серая, модификатор theme со значением normal.
