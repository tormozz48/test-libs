Описание

Блок checkbox («чекбокс») – независимый переключатель. Позволяет управлять параметром с двумя состояниями – включено и выключено. Группа чекбоксов используется для выбора более одного варианта из списка. Блок создает контейнер, содержащий нативный контрол «чекбокс» <input type="checkbox" name="имя" атрибуты>. Сам нативный контрол скрыт внутри контейнера, используется только его функциональность. Нестандартное островное оформление чекбокса задается контейнеру. В MSIE8 и ниже checkbox деградирует до нативного чекбокса <input type="checkbox" атрибуты>.

Элементы блока

  • box – добавляется в BEMHTML-шаблоне блока и трансформируется в тэг <span>. Используется для отрисовки чекбокса (нативный чекбокс скрыт).
  • control – добавляется на уровне шаблонизатора блока к содержимому checkbox. В HTML преобразуется в нативный контрол чекбокс <input type="checkbox" name="имя" атрибуты>. Служит для использования функциональности нативного чекбокса.
  • tick – элемент для отображения «галочки» в поле чекбокса. В HTML трансформируется в тэг <i>. В CSS-свойстве элемента background находится векторное изображение «галочки» в формате svg.

Объявление блока на странице

BEMJSON для использования блока:

{
    block: 'checkbox', // имя блока
    mods: {size: 'm'},  // поле модификаторов. Модификатор размера - обязательный
    checkboxAttrs: {id: 'nn', value: 'val', name: 'text', tabindex: 1}, // атрибуты чекбокса
     content: {elem: 'label', content: 'label'} // метка чекбокса
}

checkboxAttrs, content – не обязательные поля BEMJSON блока.

BEMHTML-шаблон трансформирует checkbox в HTML-тэг <span>, реализующий кастомный чекбокс, внутри которого будет находиться нативный чекбокс <input type="checkbox" атрибуты>.

Атрибуты блока

  • {Object} checkboxAttrs – поле атрибутов. Допустимые атрибуты: name, value, accesskey, tabindex, id, autofocus. Совпадают с HTML-атрибутами нативного чекбокса, который лежит в основе реализации блока checkbox.
    • {String} name – имя контрола для идентификации обработчиком формы. У чекбоксов, объединенных в группу, имена должны совпадать.
    • {String} value – значение, отправляемое на сервер при передаче формы.
    • {String} id – уникальный идентификатор чекбокса. Если id явно не задан во входных данных checkboxAttrs:{id: 'nnn'}, то будет сгенерирован автоматически. Служит для связи с меткой.
    • {Number} tabindex – определяет последовательность перехода между элементами при нажатии на кнопку Tab.

Содержимое блока

  • {Object} content – содержимое чекбокса (метка). Для создания метки (текста) контрола служит элемент блока checkbox – label. Не обязательный элемент. Текст метки задается в контенте элемента label.
{
    block: 'checkbox',
    mods: {size: 'm'},
    content: {elem: 'label', content: 'текст метки'} //добавление метки чекбоксу
}

BEMHTML-шаблон преобразует label в HTML-тэг <label>, который устанавливает связь между меткой и чекбоксом. Такая связь используется для изменения состояния чекбокса при нажатии курсором мыши или пробелом на метку. Чекбокс связывается с меткой идентификатором id внутри нативного чекбокса и указанием его значения в атрибуте for тэга <label>.

Варианты использования checkbox

  • Для управления одним параметром.
{
    block: 'checkbox',
    mods: {size: 'm'},
    checkboxAttrs: {id: 'login'},
    content: {elem: 'label', content: 'запомнить меня'}
}
  • Группа чекбоксов. Значение атрибута name у чекбоксов в группе должно быть одинаковое.
{
    block: 'checkbox',
    mods: {size: 'm'},
    checkboxAttrs: {id: 'nnn', value: 'val_1', accesskey: 'kj', name: 'text', tabindex: 1}
},
{
    block: 'checkbox',
    mods: {size: 'm', checked: 'yes'},
    checkboxAttrs: {value: 'val_2',  name: 'text', tabindex: 2}
},
{
     block: 'checkbox',
    mods: {size: 'm', disabled: 'yes', checked: 'yes'},
     checkboxAttrs: {id: 'nn', value: 'val_3', name: 'text', tabindex: 3},
     content: {elem: 'label', content: 'label'}
}

Модификаторы

Темы оформления

По умолчанию для чекбокса реализована одна тема оформления – обычная (модификатор theme со значением normal).

Размеры

Для чекбокса реализованы два размера – s и m, которые определяют размер шрифта метки и поля для проставления галочки. Модификатор размера size – обязателен при объявлении блока, по умолчанию не задан.

NB Если модификатор размера не указан во входных данных блока – чекбокс не отобразится на странице.

Таблица размеров блока

Размер блока Размер шрифта Размеры поля чекбокса
s 13px width: 12px; height: 12px;
m 15px width: 14px; height: 14px;

При необходимости вы можете доопределить нужный размер на уровне проекта. Кастомный размер.

Состояния

Модификаторы состояния чекбокса:

  • Выбран checked – характеризует активное (выбранное) состояние чекбокса.
  • Не активен disabled – чекбокс отключен.
  • В фокусе focused – чекбокс находится в фокусе.

Если во входных данных не задан модификатор состояния блока, то checkbox по умолчанию будет активен (можно нажать на чекбокс) и не выбран. В состоянии не активен (модификатор disabled со значением yes) чекбокс и метка будут недоступны для действий пользователя. Чекбокс может находиться одновременно в нескольких состояниях. Например, не активен и выбран. В состоянии в фокусе реагирует только на клик левой кнопки мыши или на нажатие пробела. Переход по чекбоксам осуществляется клавишей Tab, установка состояния выбран/не выбран – Space.

{
    block: 'checkbox',
    mods: {size: 'm', disabled: 'yes', checked: 'yes'},
    content: {elem: 'label', content: 'label'}
}

Значения по умолчанию

  • Тема оформления: обычная (модификатор theme со значением normal).
  • Состояние: активен, не выбран.
  • Атрибут id генерируется по умолчанию, если явно не задан в BEMJSON.
