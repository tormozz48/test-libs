Описание

Блок check-button («чекбаттон») – независимый переключатель («чекбокс») в виде кнопки, наследуется от блока checkbox. Используется по аналогии с обычным чекбоксом – для управления параметром с двумя состояниями – включено (кнопка вжата) и выключено (кнопка отжата) или для выбора нескольких вариантов из списка (группа чекбаттонов). В блоке check-button используется функциональность нативного контрола «чекбокс», который скрыт. В MSIE8 и ниже чекбаттон деградирует до нативного чекбокса <input type="checkbox" атрибуты>.

Элементы блока

  • control – добавляется по умолчанию на уровне шаблонизатора блока (мода default) к содержимому блока. В HTML преобразуется в нативный контрол чекбокс <input type="checkbox" name="имя" атрибуты>. Служит для использования функциональности нативного чекбокса.

  • text – обрамляет текст чекбаттона, используется в нажатом состоянии кнопки для сдвига текста без смещения рядом стоящего текста. Преобразуется BEMHTML-шаблоном в тэг <span>.

  • icon – используется для отображения иконки в чекбаттоне, рядом с текстом или самостоятельно (требует указания модификатора only-icon).

Объявление блока на странице

BEMJSON для использования блока:

{
    block: 'check-button', // имя блока
    name: 'test',          // имя контрола для его идентификации обработчиком формы
    value: '1',            // значение, отправляемое на сервер при выборе контрола
    mods: {size: 's'},   // модификаторы блока. Модификатор размера – обязательный
    content: '1'           // текст кнопки
}

В результате BEMHTML-преобразований блок трансформируется в <label>, реализующий кастомный чекбокс, внутри которого будет находиться нативный чекбокс с заданными значениями атрибутов.

Атрибуты блока

Допустимые атрибуты чекбаттона: name, value, tabindex, id – совпадают с HTML-атрибутами нативного чекбокса, который лежит в основе реализации блока.

name, value, tabindex – не обязательные, задаются в одноименных полях или поле controlAttrs входного BEMJSON блока check-button.

  • {String} name – имя чекбаттона для его идентификации обработчиком формы. Обязательное поле.
  • {String} value – значение, отправляемое на сервер при выборе контрола. Если значение value явно не задано и content не пустой – будет сгенерирован HTML-атрибут value со значением из content.

  • {String} tabindex – определяет последовательность перехода между элементами при нажатии на кнопку Tab.

  • {String} id – уникальный идентификатор чекбаттона. По умолчанию значение id генерируется автоматически. Служит для связи <label>(метка) и нативного чекбокса <input type="checkbox">.

Метка <label for="значение_id"> связывается с контролом <input type="checkbox" id="значение_id"> через свой атрибут for и аналогичным значением id у input.

Остальные HTML-атрибуты для блока не устанавливаются (например, accesskey, autofocus).

Содержимое блока

  • {String} content – содержимое чекбаттона (текст). На уровне BEMHTML-шаблона помещается в элемент блока text (преобразуемый в тэг <span>). Если content отсутствует или содержит пустое значение, кнопка отобразится без содержимого с размерами, заданными в модификаторе size.

Кнопка чекбаттона, помимо текста, может содержать текст с иконкой или только иконку.

Для отображения иконки в BEMJSON блока нужно:

  • добавить модификатор only-icon со значением yes (только для иконки без текста).
  • смиксовать элемент чекбокса icon с блоком иконки image.
{
    block: 'image',
    mix: [{block: 'check-button', elem: 'icon', mods: {'16': 'bus'}}],
    alt: 'Bus'
}

image – блок из библиотеки islands-components, служащий контейнером для изображения (преобразуется в HTML-тэг <img>).

Изображение иконки задается c помощью модификатора элемента icon блока check-button (произвольная пара «ключ:значение», в CSS которого находится путь к изображению).

Модификаторы

Темы оформления

Для check-button реализованы две темы оформления:

  • Обычная, модификатор theme со значением normal (используется по умолчанию).
{
    block: 'check-button',
    value: 'm',
    mods: {size: 'm'},
    content: 'Размер M'
}

  • Псевдо-чекбаттон, модификатор pseudo со значением yes.
{
    block: 'check-button',
    mods: {size: 'm', pseudo: 'yes'},
    content: 'Псевдо-чекбокс размером M'
}

Псевдо-чекбаттон При наличии модификатора pseudo со значением yes по умолчанию устанавливается тема псевдо (модификатор theme со значением pseudo).

Чекбокс-кнопкой с темой pseudo выглядит как прозрачная кнопка и визуально может быть вдавленной. Текст сдвигается на 1px вниз при нажатии. У псевдо-чекбокса кнопкой значение value пусто.

У псевдо-кнопки (кнопка с темой pseudo) – текст не сдвигается при нажатии, кнопка не остается во вдавленном положении.

Тема псевдо (модификатор theme со значением pseudo) может использоваться и без модификатора pseudo.

Размеры

Для check-button реализованы два размера – s и m. Модификатор размера size – обязательный при объявлении блока. Кнопка чекбаттона с темой псевдо (pseudo) по высоте меньше, чем обычная, с темой normal.

Состояния блока

  • Выбран checked – активное состояние чекбаттона, чекбаттон выбран (кнопка вжата).
  • Не активен disabled – кнопка чекбаттона не активна и нажать на нее нельзя.
  • В фокусе focused – кнопка чекбаттона находится в фокусе.
  • Нажат pressed – визуальное нажатие кнопки, чекбаттон не выбран.

Чекбаттон может находиться одновременно в нескольких состояниях. Например, не активен + выбран. В нажатом состоянии текст в кнопке чекбаттона смещается на 1px вниз.

Переход по чекбаттонам в группе, в состоянии в фокусе одного из них, может осуществляться клавишей Tab. Установка состояния выбран/не выбран – Space.

Значения по умолчанию

  • Тема оформления кнопки чекбаттона: обычная (модификатор theme со значением normal).
  • Состояние: активен, не выбран.
  • Атрибут id генерируется автоматически.
