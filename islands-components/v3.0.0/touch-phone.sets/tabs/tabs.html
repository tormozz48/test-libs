Табики

Внимание! Блок имеет нестабильный API

Блок, инкапсулирующий в себе различные варианты контролов, используемых для реализации табиков. Он предоставляет одинаковые JavaScript-API и BEMJSON-API для всех реализаций контролов, чтобы можно было легко переходить с одной на другую без переписывания кода.

Реализации контролов

В зависимости от значения модификатора _control этот блок имеет различные реализации. Если блоку передать любые модификаторы кроме _control, они будут переданы во внутренный контрол. При этом нужно самостоятельно позаботиться о разрешении зависимостей.

В виде меню (tabs_control_menu)

Внутри используется блок tabs-menu (см. пример). Переданный content (см. bemjson примера) в блок tab будет подставляться в content блока tabs-menu. При этом все элементы и модификаторы будут находиться в контектсе tabs-menu, а не к tabs.

Для блока tabs-menu никакие модификаторы поумолчанию не выставляются, нужно самостоятельно передать модификаторы _size, _theme и _layout и добавить зависимости.

В виде кнопок (tabs_control_buttons)

Внутри используется блок radio-button (см. пример).

Переданные в content (см. bemjson примера) элементы tab будут сконвертированы в элементы radio-button__radio. В контексте элементов tab можно передать любые параметры, специфичные для radio-button__radio. Если controlAttrs не будут переданы, то в качестве value радиокнопки будет автоматически выстален его порядковый номер (начинается с 0).

Так же в bemjson-е блока tabs можно передать name для radio-button, иначе он будет сгенерирован автоматически.

Для блока radio-button никакие модификаторы поумолчанию не выставляются, нужно самостоятельно передать модификатор _size и добавить зависимости. Модификатор _theme_normal выставляется самим блоком, его можно переопределить, если нужно.

Своя реализация

Можно (и нужно) легко сделать свою реализацию контрола табиков. Для этого нужно на уровне BEMHTML реализовать преобразование исходного BEMJSON в необходимый, а в JavaScript реализовать специальный API (заглушку для него можно посмотреть в tabs.js).

Связь с блоком tabs-panes

tabs-panes - это отдельный блок, реализующий панельки для табиков. Этот функционал бын вынесен в отдельный блок для того, чтобы можно было разнести по разным DOM-узлам контрол и панельки табиков для удобства верстки. Чтобы tabs-panes переключал панельки в зависимости от нашего tabs, нужно использовать поля id и panes в этих блоках соответственно:

{
    block: 'tabs',
    panes: 'oh-my-tabs-id'
    // ...
},
{
    block: 'tabs-panes',
    id: 'oh-my-tabs-id'
    // ...
},

В BEMHTML эти поля раскрываются в примешивание блока tabs-panes со связью через одинаковые id в поле js. Более подробно можно посмотреть в примерах.
